### What it does
The `ipax-calc` vendor module provides a multi-metric accessibility scoring engine that combines WCAG 2.0, APCA, and custom ergonomic heuristics. It calculates a final "IPAX" score by evaluating color contrast against biological factors like jitter, chromostereopsis, and H-K effects.

### Public interface
*   **`window.IPAX.ColorMath`**
    *   `hexToOklch(hex: string): { l: number, c: number, h: number, hex: string } | null`
    *   `relativeLuminanceWCAG(hex: string): number`
    *   `calculateWcagContrast(txtY: number, bgY: number): number`
*   **`window.IPAXScore`**
    *   `getIPAXscore(textHex: string, bgHex: string, forceContext: boolean | null = null, fontWt: number = 400): Object`
        *   Returns a JSON object containing the final score, WCAG/APCA compliance data, and a breakdown of ergonomic penalties and rewards.

### Key invariants
*   **WCAG 2.0 Isolation**: `relativeLuminanceWCAG` must be computed directly from sRGB; it is explicitly prohibited to derive WCAG luminance from OKLCH values to ensure standard-compliant results.
*   **Compliance Clamping**: Rewards cannot inflate a score across a legal compliance grade boundary (e.g., a score of 1 cannot be boosted to 2, and 2 cannot be boosted to 3).
*   **Integer Compliance Floor**: The base `complianceLevel` is always derived from the floor of the minimum of WCAG and APCA scores, ensuring the baseline remains a whole number before ergonomic adjustments.

### Non-obvious decisions
*   **OKLCH for Ergonomics**: While WCAG and APCA are calculated via their respective standard pipelines, the system converts colors to OKLCH specifically for "biological" modeling (jitter, H-K effect). This is done because these phenomena lack standardized algorithms, and OKLCH provides a perceptually uniform space suitable for modeling these specific visual effects.
*   **Biological Notation**: The `ipaxString` output uses a custom format `X(Y)` (e.g., `2(3)`) to denote when the biological score is lower than the legal compliance score, explicitly surfacing the discrepancy between regulatory requirements and ergonomic performance.
*   **Reward/Penalty Gating**: The H-K (Helmholtz-Kohlrausch) effect penalty is gated strictly to the "Excellence Zone" (`absLc >= 75`). This implies the system prioritizes basic legibility (contrast) over secondary visual effects until a high threshold of contrast is achieved.
